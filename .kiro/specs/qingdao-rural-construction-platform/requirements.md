# 需求文档

## 项目介绍

青岛市农房建设管理和乡村建设工匠培训、信用考核评价信息平台是一个综合性的数字化管理系统，旨在贯彻落实国家对乡村建设工匠培训和管理的重要指示精神。该平台将建立完善的农房建设管理体系和乡村建设工匠培训管理体系，覆盖青岛市7个区市84个街道，通过数字化手段提升农房建设管理效率和工匠培训质量。

## 需求

### 需求 1 - 农房信息管理系统

**用户故事：** 作为农房管理人员，我希望能够全面管理农房基础信息，以便实现农房建设的数字化监管。

#### 验收标准

1. 当用户访问系统时，系统应当显示青岛市农村建房一张图，包含区市边界信息
2. 当用户查询农房信息时，系统应当提供房屋属性信息，包括地址、建筑时间、房屋层数、房屋高度等详细信息
3. 当农房建设过程中时，系统应当支持建设过程照片录入和验收信息录入
4. 当需要数据导入时，系统应当支持与农业部门对接，自动导入每年经过审批的新建、改建、扩建农房基础信息
5. 当查询历史数据时，系统应当包含近五年来农村低收入困难群体危房改造信息

### 需求 2 - 乡村建设工匠管理系统

**用户故事：** 作为工匠管理人员，我希望建立完整的工匠管理名录和培训体系，以便规范化管理乡村建设工匠。

#### 验收标准

1. 当建立工匠档案时，系统应当记录工匠基本信息、培训情况、工程业绩、技能等级认定、奖罚情况、信用评级
2. 当管理工匠队伍时，系统应当支持工匠组建的施工班组、合作社和合伙制企业等工匠队伍情况管理
3. 当进行工匠培训时，系统应当支持上传乡村建设工匠培训资料，录入培训证明
4. 当培训管理时，系统应当确保每年度培训不低于40个学时，其中线下授课不低于24个学时
5. 当需要培训统计时，系统应当能够定期导出乡村建设工匠培训等级信息

### 需求 3 - 工匠信用评价系统

**用户故事：** 作为信用管理人员，我希望建立动态的工匠信用评价体系，以便形成良性市场竞争和正向激励机制。

#### 验收标准

1. 当评价工匠信用时，系统应当建立完整的信用评价体系，包含加分减分事项
2. 当信用变动时，系统应当支持动态更新加分减分事项
3. 当公示信用信息时，系统应当向社会公布工匠相关信息
4. 当农户选择工匠时，系统应当提供信誉良好、技术过硬的工匠推荐
5. 当进行信用评级时，系统应当定期更新工匠信用等级

### 需求 4 - 质量安全监管系统

**用户故事：** 作为质量监管人员，我希望实时监控农房建设质量和安全，以便确保建设标准和安全要求。

#### 验收标准

1. 当农房建设时，系统应当支持录入六到场信息（勘察、设计、施工、监理、建设、质监到场）
2. 当日常检查时，系统应当支持区市、镇街质量安全检查信息录入
3. 当收集反馈时，系统应当支持群众满意度反馈收集和分析
4. 当进行抽样调查时，系统应当支持村民问卷调查，包括新建、改建、扩建农房建房满意度和农村危房改造满意度测评
5. 当现场检查时，系统应当支持农房问题拍照上传和现场检查记录

### 需求 5 - PC端数据采集工具

**用户故事：** 作为区市、镇街、村庄工作人员，我希望使用便捷的PC端工具进行数据填报，以便高效完成集中数据录入工作。

#### 验收标准

1. 当进行数据填报时，系统应当为每个村庄配置对应的数据填报端口
2. 当填报流程时，系统应当将所需填报数据项进行流程化管理
3. 当用户操作时，系统应当结合一线工作习惯，提供直观易用的操作界面
4. 当数据录入时，系统应当支持批量数据导入和编辑功能
5. 当权限管理时，系统应当根据用户角色提供相应的数据访问和编辑权限

### 需求 6 - 移动端小程序应用

**用户故事：** 作为市级、区市、镇街、工匠、农户用户，我希望通过手机小程序随时随地进行相关操作，以便提高工作效率和便民服务水平。

#### 验收标准

1. 当工匠使用时，系统应当支持工匠录入建房情况和施工进度
2. 当农户使用时，系统应当支持农户申请建房手续和查询审批进度
3. 当镇街管理时，系统应当支持后台审批功能和日常现场检查情况录入
4. 当区市监管时，系统应当支持通过小程序抽查部分农房建设情况
5. 当现场作业时，系统应当内置农房位置信息，支持现场拍照上传和数据同步到PC端后台

### 需求 7 - 系统集成和数据管理

**用户故事：** 作为系统管理员，我希望系统具备良好的集成能力和数据管理功能，以便确保数据准确性和系统稳定性。

#### 验收标准

1. 当系统部署时，系统应当基于PostgreSQL数据库，使用Next.js、Vite、pnpm技术栈
2. 当界面设计时，系统应当采用Ant Design、Tailwind CSS、shadcn组件库确保界面一致性
3. 当数据对接时，系统应当支持与农业部门系统对接，实现数据自动同步
4. 当数据备份时，系统应当提供完整的数据备份和恢复机制
5. 当系统维护时，系统应当提供日志记录、性能监控和错误处理功能
